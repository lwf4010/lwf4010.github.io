上午学习了在虚拟机中使用和配置桥接模式，了解ip的分类，以及在网络中寻找网站的真实ip，下午学习了搭建博客网站并且链接GitHub。

一.计算机网络

1.桥接模式和nat

桥接模式：可以让虚拟机和物理机在同一网络ip下，在桥接模式下，虚拟机开启网络服务，不在同一网络下的其他主机也可以访问到。
如何配置桥接模式？
1.打开虚拟机，在虚拟机上点击右键，选择设置。并且在设置中开启网络桥接模式

2.点击vmware，左上角编辑，选择虚拟网络编辑器，点击桥接模式，如果连着wifi要选择网卡。

nat：由物理机分出ip，虚拟机要访问网络时必须通过物理机网络ip出去，其他人无法反向访问虚拟机。

2.内网与公网

1.内网
可以通过路由器分配ip，可以作为独立的网络系统工作。如校园网
a类：10.0.0.010.255.255.255
b类：172.16.255.255172.31.255.255
c类：192.168.0.0~192.168.255.255
2.公网
ip唯一。如百度公司的ip
3.其他
子网掩码
127.0.0.0/8
ipv4和ipv6
ipv4少但是简单好记
ipv6多但是难记并且大公司使用的少
特殊ip
255.255.255.255 有限广播域
0.0.0.0 任意地址
127.0.0.0/8 回环地址
169.254.0.0/24本地链路地址

二.nmap工具

nmap是kali中的工具
打开kali终端
nmap -h可以调出帮助，可以用ai翻译
使用nmap nmap -p 1-xxx（端口） 要查询的ip
端口最多65536个
0-1023紧密绑定与一些企业的服务，如80是http服务，21是ftp服务，443是htpps服务。

三.搭建博客

1.下载软件

  1. 提前安装好(node.js)[https://nodejs.org/en] 左边的那个
  2. 提前安装好(GIt)[https://git-scm.com/downloads]
  3. 提前注册好一个github账号
  4. 下载在D:\New Folder下

2.博客的搭建

  1. 在本地找个目录放博客源文件，路径不要有中文
  2. 在选定的目录中，右键Git Bash Here
  3. 使用npm命令安装Hexo，输入：

npm install -g hexo-cli //安装hexo
hexo init blog //初始化一个名为blog的项目
cd blog //进入blog
hexo g //编译构建

检测
hexo new test1 //博客新建一个文章
hexo g //构建
hexo s //启动服务
之后就可以访问127.0.0.1:4000访问blog了

3.利用github搭建hexo博客

配置ssh

  1. cd ~/.ssh
  • 如果提示：No such file or directory 说明你是第一次使用git。

2.ssh-keygen -t rsa -C “邮件地址”,连续三次回车最终 会生成一个文件在用户目录下，打开用户目录到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容，打开你的github主页，进入个人设置 -> SSH and GPG keys -> New SSH key：
3.key填写C:\Users\Administrator.ssh\id_rsa.pub中内容
尝试ssh连接,测试是否成功

ssh -T git@github.com # 注意邮箱地址不用改

如果提示Are you sure you want to continue connecting (yes/no)?，输入yes，然后会看到下面这个，就算成功：

Hi liuxianan! You’ve successfully authenticated, but GitHub does not provide shell access.

4.输入npm install hexo-deployer-git –save
在上一个Git Bash Here还需要继续配置

git config –global user.name “luozhixiaowo”// 你的github用户名，非昵称
git config –global user.email  “xxx@qq.com“// 填写你的github注册邮箱
